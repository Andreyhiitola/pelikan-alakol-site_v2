## Как добавить администраторов

### Шаг 1: Узнать Telegram ID администратора

**Администратор должен:**
1. Открыть Telegram
2. Найти бота: @userinfobot
3. Написать `/start`
4. Бот покажет: `Id: 123456789` ← это Telegram ID

### Шаг 2: Добавить ID на VPS

**Подключиться к VPS:**
```bash
ssh root@85.192.40.138
cd ~/pelikan-bot
```

**Открыть .env:**
```bash
nano .env
```

**Добавить ID через запятую:**
```bash
# Было:
ADMIN_IDS=31310268

# Стало (несколько админов):
ADMIN_IDS=31310268,987654321,555666777
```

**Сохранить:** Ctrl+O, Enter, Ctrl+X

**Перезапустить бота:**
```bash
docker-compose restart bot
```

✅ Новые админы начнут получать уведомления!

## Работа с заказами

### Принять заказ (по умолчанию)

Заказ автоматически в статусе `принят` после оформления.

### Изменить статус на "Готовится"
```
/update 1234 готовится
```

Клиент получит:
```
🔔 Статус заказа изменён

📋 Заказ #1234
Статус: Готовится 👨‍🍳

Примерное время ещё ~10 минут
```

### Изменить статус на "Готов"
```
/update 1234 готов
```

Клиент получит:
```
🎉 Ваш заказ готов!

📋 Заказ #1234
Можете забрать в баре 🍽️

💳 Оплата: 3800₸
```

### Изменить статус на "Выдан"
```
/update 1234 выдан
```

Заказ переходит в архив и не показывается в `/orders`

### Отменить заказ
```
/update 1234 отменён
```

Клиент получит уведомление об отмене.

## Что делать если...

### Клиент не пришёл за заказом

1. Проверить контакт:
```
   /orders - посмотреть Telegram username
```

2. Написать клиенту:
```
   @ivan_user, ваш заказ #1234 готов,
   ждём вас в баре!
```

3. Если не отвечает 30 минут:
```
   /update 1234 отменён
```

### Telegram клиента не указан

**В уведомлении админу будет:**
```
📋 #1234
👤 Иван
🏨 Комната: 205
📱 Telegram: не указан ❌

💡 Позвонить в комнату когда готово!
```

**Действия:**
1. Приготовить заказ
2. Позвонить в комнату 205
3. Или написать записку и отправить с персоналом

### Заказ срочный
```
/update 1234 готовится
```

Сразу начать готовить, уведомить клиента что заказ в работе.

### Ингредиента нет

1. Отменить заказ:
```
   /update 1234 отменён
```

2. Написать клиенту:
```
   К сожалению, бургер временно недоступен.
   Можем предложить альтернативу: [список]
```

## Troubleshooting

### Бот не отвечает
```bash
ssh root@85.192.40.138
cd ~/pelikan-bot

# Проверить статус
docker-compose ps

# Если не работает - перезапустить
docker-compose restart bot

# Посмотреть логи
docker-compose logs -f bot
```

### Заказы не приходят с сайта
```bash
# Проверить webhook
curl http://85.192.40.138:8080/health

# Должно вернуть: {"status":"ok"}

# Если нет - проверить порт
sudo ufw status
sudo ufw allow 8080/tcp
```

### База данных повреждена
```bash
# Backup текущей БД
cp data/orders.db data/orders.db.backup

# Проверить БД
sqlite3 data/orders.db "PRAGMA integrity_check;"

# Если ошибки - восстановить из backup
cp data/orders.db.backup data/orders.db
docker-compose restart bot
```

### Заказ пришёл без Telegram-данных (с сайта)

Если гость оформил заказ через сайт не из Telegram — Telegram username не будет. В уведомлении придёт:
```
📋 #1234
👤 Иван
🏨 Комната: 205
📱 Telegram: не указан ❌
```

Позвонить в комнату: **+7 (72833) 3-00-02** (ресепшн может соединить).

2. Задеплоить на сервер — меню обновится автоматически
